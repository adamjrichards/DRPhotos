History {hist}
07-04-23: reconstructed from lists of scanned files for trays 89d1, 89d2. used FRN.SCR to rename scanned files to new prefix + last 3 characters (scanning sequence number = tray slot number). Two tag lists obtained from GV.SCR. These merged, tray 1 first, tray2 second. The combined list sorted column 9, the result forming the basis of this portfolio file.


Scanning was done 4 slides at a time, with auto-numbering starting at the beginning tray slot number. This allows scanning 44 to 50 per hour, but doesn't allow cropping, rotation, or specifying the actual picture number. A scripted operation could probably be devised to allow speedy renaming from tray number to actual picture number. While the slides are stored in the Carousel projector trays, it would probably be a good idea to retain both numbers on the "gra" tag for easy reference.

A few scans appear to be out-of-focus. May have to be redone. Seems like a software or hardware fault in the scanner. Try exiting and reloading the scanner application for each group of 4.

07-05-01: finished cutting up sound clips. Still needs picture edit. Redo levels after cropping. t89d1041 appears to be wrong -- check slide tray. Mark the worst for rescanning, if badly focussed. Images were tray-numbered to speed up scanning, should eventually be renumbered according to actual image numbers. Could be done by adding the image numbers to the last field of each "gra" tag then using bmp.scr "z" option 6.

07-05-18: finished picture edits yesterday. Deferred dust spotting til later. Concentrated on cropping, one-click fixes using macros to adjust levels, colour, sharpness. This goes much faster and may be suitable for whimsical slide shows like this.

11-03-14: moved files to t89d folder. Main files, THMs, and TIFs can't be in a SHOWS subfolder if these are to be kept thinly populated for portable use. Keeping thumbnails in the SHOWS subfolders is only needed if stand-alone WWWB versions containing only automated shows are to be created. These folders on the C-drive which are used to create distribution, view only versions of the system, should contain only the PTF file and any sound files.
